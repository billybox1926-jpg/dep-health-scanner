dep-health-scanner — Architecture

Python implementation (current shipped state)

Entry: depscan.py

CLI entry script. Adds src/ to sys.path, imports Typer app, and launches CLI.

▾

cli.py — Typer App

Command router. Subcommands: scan, stats. Parses flags and invokes scanner pipeline.

lockfile.py — LockfileDetector

Auto-detects lockfiles (package-lock.json, Cargo.lock, Pipfile.lock, go.mod) in target path.

▾

models.py — Data model

Dependency, Ecosystem (npm/cargo/pip/go), Vulnerability, ScanResult.

scanner.py — Scanner

Parallel scan using ThreadPoolExecutor. Coordinates RegistryClient + VulnerabilityClient.

▾

RegistryClient

Fetches latest package versions from npm / crates.io / Go module proxy. Reads/writes Cache.

VulnerabilityClient

Queries OSV API for known vulns. Reads/writes Cache.

reporter.py — Reporter

Rich terminal output: summary tables, severity colors, optional --format json / text.

▾

cache.py — SQLite Cache

Persistent local cache at ~/.cache/dep-health-scanner/cache.sqlite. Stores: registry_versions + vulnerabilities tables. Thread-safe via per-connection (check_same_thread=False) + context-managed queries.
